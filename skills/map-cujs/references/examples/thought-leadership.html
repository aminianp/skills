Steps (job-oriented)

  1. Land. User arrives at the blog index from the home nav, from the Discovery CUJ, or directly from a shared link.
  2. Skim list. Titles, short excerpts, and tag chips are visible together. The user is asking: "is anything here likely to teach me something?" If they have a specific topic in mind, they may click a tag chip to narrow the list (per FAQ Q6 resolved — tags are inline filter affordances, not a separate mode).
  3. Pick a post. The user picks based on which post seems most likely to deliver on their interest — a combination of title sharpness, excerpt clarity, and tag match. Tag-narrowing is part of this step, not a strategic pre-decision.
  4. Read and form an opinion. User engages with the post, looking for depth (does this go beyond table-stakes?), clarity (is the thinking well-structured?), and originality (does this say something they haven’t already heard?). Opinion forms during reading. The exit varies:
    • Peer: may follow, subscribe (future), share, return for more, or move on.
    • Recruiter: takes the depth signal back to their fit evaluation.

  1. Empty tag filter.
    • Type: Functionality (primary) + UX (empty-state design)
    • Addressed by: implementation-brief (only render tags with at least one post) + design-wireframes (empty-state with adjacent-tag suggestions)
    • Trigger: user clicks a tag chip and lands on an empty post list.
    • User experience: conclusion drawn in seconds: "the blog claims to cover this but doesn’t" — lowers trust in the blog’s depth and discourages further filtering.
    • Recovery: only render tag chips for tags that have at least one post. If a deep-link to an empty tag exists (e.g., from a shared old URL), show suggested adjacent tags rather than an empty list.
  2. Title or excerpt sets wrong expectation.
    • Type: Content
    • Addressed by: editorial (pre-publish title-vs-content QA gate)
    • Trigger: the post title sounds like one thing; the content delivers another. Common cause: cute titles that prioritize cleverness over clarity.
    • User experience: reader skims, realizes the mismatch, bounces. Comes away with a lower opinion of the writer.
    • Recovery: titles describe what the post actually delivers; excerpts preview a real takeaway from the post, not just the opening sentence.
  3. Stale posting cadence.
    • Type: Content
    • Addressed by: editorial (cadence schedule + evergreen curation when gaps occur)
    • Trigger: newest visible post is 6+ months old.
    • User experience: visitor concludes Pouyan stopped writing — and stops checking back. The conclusion sticks even after new posts get published.
    • Recovery: dates visible on the index. If a real cadence gap exists, consider de-emphasizing dates and curating "evergreen" posts as the primary entry surface; revisit when posting resumes.

    • What changed: Removed the Strategy decision node and the Filter / Browse alternative paths. The happy path is now Land → Scan → Pick → Read; tag-narrowing is a sub-action inside Scan rather than a separate strategic step. Wireframe implication: tag chips inline with the post list.
