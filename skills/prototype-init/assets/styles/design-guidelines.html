Reference

Vendor Design Guidelines

Canonical, vendor-published design guidance for every form factor a project might target. Each row pairs a target slug (used in prototype/APPROVED as target: <slug>) with the official source for that platform’s conventions. Use it when picking a target, when reviewing a prototype against platform norms, or when handing off to engineering.

All links go to the vendor’s own design site. No aggregator pages, no third-party reposts.

Phone

Apple

iOS

target: phone-ios
developer.apple.com · Designing for iOS ↗

System fonts, controls, navigation patterns, gestures, and Apple-specific technologies (Sign in with Apple, Live Activities, App Clips). Part of the Human Interface Guidelines.

Google

Android (Material 3)

target: phone-android
m3.material.io ↗

Material Design 3 (“Material You”) is Google’s open-source design system for Android phones. Covers components, motion, color, typography, and adaptive layout.

Tablet

Apple

iPadOS

target: tablet-ipados
developer.apple.com · Designing for iPadOS ↗

iPad-specific guidance: split view, slide over, pointer interactions, Apple Pencil, multitasking, larger canvas patterns.

Google

Android (large screens)

target: tablet-android
m3.material.io · Applying layout: large screens ↗

Material 3 guidance for tablets and foldables: list-detail layouts, supporting panes, expanded navigation rails.

Watch

Apple

watchOS

target: watch-watchos
developer.apple.com · Designing for watchOS ↗

Apple Watch design: glanceable surfaces, Smart Stack, complications, Digital Crown interactions, Always-On treatment.

Google

Wear OS

target: watch-wear-os
developer.android.com · Wear UI Design ↗

Wear OS principles: focus on one-or-two tasks, round-screen layouts, tile + complications model, larger margins for round displays.

Apple

tvOS

target: tv-tvos

Apple TV design: focus engine, parallax artwork, top-shelf, Siri Remote interactions, distance reading.

Google

Android TV / Google TV

target: tv-android-tv

10-foot UI design: D-pad navigation, focus model, immersive lists, featured carousels, navigation drawers, app banners.

Desktop

Apple

macOS

target: desktop-macos

Mac design: window chrome, menus, sidebars, toolbars, keyboard shortcuts, pointer interactions, Mac-specific technologies (Continuity, Stage Manager).

Microsoft’s Fluent 2 design system: used across Windows, Office, and Teams. Covers components, motion, color tokens, type ramp, and React + Web Components implementations.

Terminal / CLI

Community canonical

CLI Guidelines

target: terminal
clig.dev ↗

No single vendor canonical. Command Line Interface Guidelines is the most widely-cited modern reference, distilling traditional UNIX philosophy and current best practices for arguments, flags, output, errors, and subcommands.

Cross-cutting: accessibility

W3C

WCAG 2.1

applies to all targets
w3.org/WAI/WCAG21 ↗

Web Content Accessibility Guidelines. Legally referenced in many jurisdictions. AA conformance is the common product bar.

W3C

ARIA Authoring Practices

target: web-spa
w3.org/WAI/ARIA/apg ↗

Reference implementations for common widgets (combobox, dialog, listbox, tree, etc.) with keyboard interaction and ARIA roles.

Adding a target not on this list

If your project targets a platform we haven’t covered — foldables, smart speakers, gaming consoles, e-readers, in-flight entertainment, smart home displays — find the vendor’s own design site, add a row to this page, and pick a target slug for the manifest. Don’t link to aggregator pages, third-party tutorials, or community summaries unless the vendor genuinely doesn’t publish a canonical reference.
