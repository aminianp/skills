Site Nav

Sticky top nav: site logo on the left, primary nav links + social icon row on the right. Composes nav-link and icon-link components. The active link is driven by data-active="true", set by the prototype's hash router.

Responsive: hamburger drawer below md

Below the md breakpoint (768px), the entire inline nav (primary links + separator + social icons) hides. In its place, a hamburger button appears on the right; tapping it opens a full-screen drawer overlay with the same nav items at large, tappable sizes plus the social icons pinned to the bottom.

Why a drawer, not a dropdown? Drawers cover the whole viewport, which on a phone means every link is reachable with one thumb. Dropdowns require precise tapping near the top of the screen and don't accommodate the social icons + branding header. Drawer is the canonical mobile-nav pattern (Material, iOS Mail, Slack, Notion all use it).

JS contract: #nav-toggle opens the drawer (aria-expanded=true), #nav-close + Escape + clicking any [data-nav-drawer-link] closes it, and a window-resize listener auto-closes the drawer if the user crosses back into desktop width.

<!-- Hamburger toggle (mobile only) -->
<button id="nav-toggle" class="md:hidden -mr-2 p-2 ..."
   aria-label="Open menu" aria-expanded="false"
   aria-controls="nav-drawer">
  <svg>...3-line icon...</svg>
</button>

<!-- Inline nav (md+ only) -->
<div class="hidden md:flex items-center gap-7 text-sm">
  <!-- 3 nav-links + separator + 4 icon-links -->
</div>

<!-- Mobile drawer (full-screen overlay) -->
<div id="nav-drawer" class="md:hidden fixed inset-0 z-30
   bg-bg flex flex-col" hidden>
  <div class="border-b ... flex justify-between">
    <a href="#/home" data-nav-drawer-link>pouyan.fyi</a>
    <button id="nav-close"><svg>close</svg></button>
  </div>
  <nav class="flex-1 px-6 py-10 flex flex-col gap-1">
    <a data-nav-drawer-link class="display text-3xl
       font-semibold py-3">Home</a>
    <!-- Work, Blog -->
  </nav>
  <div class="border-t px-6 py-5 flex gap-6 items-center">
    <!-- 4 icon-links at footer of drawer -->
  </div>
</div>

Touch-target sizes inside the drawer: each nav link gets py-3 (~48px tall) and text-3xl for easy thumb-reach. Icons sized to 22px (vs. 18px in the desktop inline nav).
