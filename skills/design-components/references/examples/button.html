Responsive: full-width stacked below sm

In a button group with two or more CTAs, stack vertically and fill the available width below the sm breakpoint (640px). Above sm, lay out inline. Bump button vertical padding from py-2.5 to py-3 on mobile so the touch target is at least 44px tall, the Apple HIG / Material minimum.

<div class="flex flex-col sm:flex-row gap-3">
  <a class="bg-primary text-primary-fg rounded
     px-5 py-3 sm:py-2.5
     text-sm font-medium
     hover:opacity-90 cursor-pointer
     text-center">
    See my work &rarr;
  </a>
  <a class="bg-surface border border-border rounded
     px-5 py-3 sm:py-2.5
     text-sm font-medium
     hover:border-accent hover:text-accent
     cursor-pointer
     text-center">
    Read the blog
  </a>
</div>

Why full-width? Inline buttons next to each other on a phone either overflow horizontally, wrap their text awkwardly mid-label, or get squished to unreadable widths. Stacked + full-width gives each CTA equal weight, prevents text wrapping, and matches the comfortable-thumb-reach pattern used by every native app's primary action button.

When NOT to stack: a single primary button on its own (no group) doesn't need full-width — inline-block at its natural width is fine. Stacking matters when there are two or more CTAs sharing a row.

Tokens

  • --color-primary — primary fill
  • --color-primary-fg — primary text
  • --color-surface — secondary fill
  • --color-border — secondary border (default)
  • --color-accent — secondary border + text on hover
  • --radius — corner radius (consumed via Tailwind rounded)
